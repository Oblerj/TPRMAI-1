🧪 Authentication Testing Guide

⚠️ IMPORTANT: You must clear browser cookies or use Incognito/Private windows between each user test to avoid PKCE cookie conflicts.

Testing Each User (Method 1: Clear Cookies)

Step 1: Test Mock Admin
  1. Open http://localhost:3000 in your browser
  2. Click "Sign In" button
  3. Click "Mock Admin" button on OIDC page
  4. ✅ Should redirect to dashboard successfully
Step 2: Clear Cookies Before Next Test
  1. Open browser DevTools (F12 or Cmd+Option+I)
  2. Go to "Application" or "Storage" tab
  3. Find "Cookies" → http://localhost:3000
  4. Click "Clear all cookies" or delete all cookies
  5. Also clear http://localhost:10090 cookies if present

Or use keyboard shortcut:

  • Chrome/Edge: Ctrl+Shift+Delete (Windows) or Cmd+Shift+Delete (Mac)
  • Select "Cookies" only
  • Time range: "Last hour"
  • Click "Clear data"
Step 3: Test Mock Analyst
  1. Refresh the page or go to http://localhost:3000
  2. Click "Sign In" button
  3. Click "Mock Analyst" button on OIDC page
  4. ✅ Should redirect to dashboard successfully
Step 4: Repeat for Remaining Users
  • Clear cookies again
  • Test Mock Viewer
  • Clear cookies again
  • Test Mock Vendor

Testing Each User (Method 2: Incognito Windows)

Easier Method - Use 4 Separate Incognito Windows
  1. Open 4 separate Incognito/Private windows (Ctrl+Shift+N or Cmd+Shift+N)
  2. In Window 1: Test Mock Admin
  3. In Window 2: Test Mock Analyst
  4. In Window 3: Test Mock Viewer
  5. In Window 4: Test Mock Vendor

Each incognito window has isolated cookies, so no interference!

Why This Is Necessary

The PKCE Flow Issue:

When you authenticate as Mock Admin, NextAuth creates encrypted PKCE cookies for that session. When you try to authenticate as a different user WITHOUT clearing cookies first, the old PKCE cookies from the Admin session conflict with the new authentication attempt, causing a "pkceCodeVerifier value could not be parsed" error.

This is expected OAuth/OIDC behavior - each authentication flow should start with a clean cookie state.

Expected Results

✅ Success Indicators:
  • All 4 users redirect to /dashboard after clicking their button
  • No "Server error" messages appear
  • Dashboard shows user's name and role correctly
  • Logs show: [Auth Profile] { sub: 'mock-analyst', ... }

Monitoring Logs

Open terminal and run:

tail -f /tmp/nextjs.log

Watch for successful authentication:

[Auth Profile] {
  sub: 'mock-analyst',
  email: 'analyst@app.local',
  name: 'Mock Analyst',
  role: 'ANALYST'
}
GET /dashboard 200

Quick Test Script

Test all users quickly using separate incognito windows:

  1. Open 1st incognito window → Test Admin → Leave open
  2. Open 2nd incognito window → Test Analyst → Leave open
  3. Open 3rd incognito window → Test Viewer → Leave open
  4. Open 4th incognito window → Test Vendor → Leave open

All 4 should now be simultaneously logged in as different users in different windows!
